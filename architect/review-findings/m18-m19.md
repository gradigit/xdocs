# M18+M19 Review Findings — 2026-03-09

## Adversarial Review (M18+M19 Code Changes)

### WARNING 1: _compare_manifests defeats --skip-unchanged (sync_runtime_repo.py:158)
**Confidence**: 95%
`_component_key` returns `""` for directory components without `--hash-tree`. The guard `if old_key and new_key` evaluates to `False`, so control always falls to `all_unchanged = False`. Directories (pages, meta, lancedb-index, src) always report as CHANGED.
**Fix**: Fall back to comparing `files` count and `bytes` size when no hash available.

### WARNING 2: CalVer tag collision skips .1 suffix (sync_runtime_repo.py:584)
**Confidence**: 95%
`max_n` starts at 1. When only base tag exists, `max(1, 1) = 1`, next tag = `.2`. Sequence: base, .2, .3 (skips .1).
**Fix**: Initialize `max_n = 0`.

### WARNING 3: search_pages crashes on empty query (pages.py:19)
**Confidence**: 90%
`sanitize_fts_query("")` returns `""`, FTS5 `MATCH ""` raises OperationalError.
**Fix**: Add early return for empty/whitespace-only query.

## Query Optimization Research (Codebase Analysis + Web Research)

### CRITICAL BUG: Silent TypeError in augmentation page search (answer.py:1298,1326)
`_search_pages(conn, query=..., exchange=...)` but function signature requires `url_prefix=`. TypeError caught by `except Exception` blocks. Page search augmentation for code_snippet and request_payload has **never worked**.
**Fix**: Change `exchange=` to compute `url_prefix` from exchange domain prefixes.

### CRITICAL: Dead code — query_type_hint always "question" (answer.py:724,771)
All calls to `_search_pages_with_semantic` pass `query_type_hint="question"`. RRF weight entries for code_snippet ([0.7, 1.3]) and request_payload ([1.0, 1.0]) are unreachable.
**Fix**: Propagate `classification.input_type` to `query_type_hint`.

### CRITICAL: BM25 shortcut includes "question" against design doc (answer.py:474)
Research doc at score-fusion.md:36 explicitly says "NOT question". Including "question" skips vector search for natural language queries.
**Fix**: Remove "question" from the shortcut types.

### MEDIUM: Position-aware blend scores discarded by outer RRF (semantic.py→answer.py)
`blended_score` from semantic search is lost when results are converted to DB row dicts in answer.py. Blend only indirectly affects results through rank ordering.

### MEDIUM: Synonym map gaps
Missing: futures/perpetual/perp/contract, leverage/margin, spot/cash, sandbox/testnet.
