# Research: CEX OpenAPI/Swagger Specifications
Date: 2026-02-12
Depth: Full

## Executive Summary

Of the 16 exchanges tracked in this project, **7 have official or semi-official OpenAPI/Swagger specs**, **3 have high-quality community specs**, and **6 have no known machine-readable API specification** (relying solely on HTML documentation).

Key findings:
- **Binance** has the richest spec ecosystem: official Swagger (spot only), official Postman collections (all products), and openxapi-generated OpenAPI+AsyncAPI specs covering spot, futures, options, and portfolio margin.
- **KuCoin** has the best official SDK+spec story: their Universal SDK is code-generated from official OpenAPI specs covering spot, futures, margin, earn, broker, and more.
- **Gate.io** documents their entire APIv4 following the OpenAPI specification and generates official SDKs from it, but the spec JSON endpoint returns an error when accessed directly.
- **Bitstamp** publishes an official OpenAPI spec (embedded in their Redoc docs page with a download button).
- **Upbit** has a community Swagger-generated multi-language client (uJhin/upbit-client).
- **openxapi** covers Binance (comprehensive) and OKX (REST only); other exchanges are not yet covered.
- **exchange-collection** (kanekoshoyu) covers 11 exchanges with OpenAPI+AsyncAPI specs, including Hyperliquid, Bitget, OKX, and Binance.
- No exchange publishes an official AsyncAPI spec for WebSocket APIs. The only AsyncAPI specs come from community projects (openxapi, exchange-collection).

Already imported (status quo):
- Binance: official Swagger + openxapi OpenAPI
- Bybit: official Postman collection
- KuCoin: official OpenAPI spec
- Bitget: community spec (exchange-collection)

## Exchange-by-Exchange Findings

### 1. Binance

- **Official Swagger spec**: https://github.com/binance/binance-api-swagger
  - Format: Swagger 2.0 (YAML), file: `spot_api.yaml`
  - Coverage: Spot only (`/api/*`, `/sapi/*`). Does NOT cover futures (`/fapi/*`, `/dapi/*`) or options (`/vapi/*`)
  - Last updated: October 8, 2024 (v1.22.0)
  - 164 stars, actively maintained by Binance team
  - Limitations: No futures, no CORS support for authenticated requests
  - Live Swagger UI: https://binance.github.io/binance-api-swagger/

- **Official Postman collections**: https://github.com/binance/binance-api-postman
  - Format: Postman Collection v2.1
  - Coverage: Spot, margin, futures, options, wallet, algo trading, and more
  - 1,600+ stars, 280 commits, actively maintained
  - Collections in `collections/` directory, environments in `environments/`
  - Most comprehensive official machine-readable source for Binance

- **openxapi specs** (community, AI-generated from docs):
  - REST OpenAPI 3.0:
    - Spot: https://raw.githubusercontent.com/openxapi/openxapi/main/specs/binance/openapi/spot.yaml
    - USDS-M Futures: https://raw.githubusercontent.com/openxapi/openxapi/main/specs/binance/openapi/umfutures.yaml
    - COIN-M Futures: https://raw.githubusercontent.com/openxapi/openxapi/main/specs/binance/openapi/cmfutures.yaml
    - Options: https://raw.githubusercontent.com/openxapi/openxapi/main/specs/binance/openapi/options.yaml
    - Portfolio Margin: https://raw.githubusercontent.com/openxapi/openxapi/main/specs/binance/openapi/pmargin.yaml
  - AsyncAPI 3.0 (WebSocket + Streams):
    - Spot WS: https://raw.githubusercontent.com/openxapi/openxapi/refs/heads/main/specs/binance/asyncapi/spot.yaml
    - Spot Streams: https://raw.githubusercontent.com/openxapi/openxapi/refs/heads/main/specs/binance/asyncapi/spot-streams.yaml
    - USDS-M Futures WS: https://raw.githubusercontent.com/openxapi/openxapi/main/specs/binance/asyncapi/umfutures.yaml
    - (Plus cmfutures, options, pmargin variants)
  - Sub-products covered: Spot Trading, Margin Trading, Algo Trading, Wallet, Copy Trading, Convert, Sub Account, Binance Link, Futures Data, Portfolio Margin Pro, Staking, Dual Investment, Mining, Crypto Loan, VIP Loan, C2C, Fiat, NFT, Gift Card, Rebate, Simple Earn, Binance Pay History
  - Quality: Generated by Go program parsing official docs; includes per-endpoint YAML files

- **exchange-collection spec** (community):
  - REST: https://raw.githubusercontent.com/kanekoshoyu/exchange-collection/refs/heads/main/asset/binance_rest_openapi.yaml
  - WS: https://raw.githubusercontent.com/kanekoshoyu/exchange-collection/refs/heads/main/asset/binance_ws_asyncapi.yaml

- **Notes**: Binance has the most complete spec coverage of any exchange. The openxapi specs are the most comprehensive single source, covering 22+ sub-products. For the project, the Postman collections could be valuable for importing endpoint data for new sections (options, wallet, copy_trading, etc.).

### 6. Gate.io

- **Official OpenAPI spec**: Gate.io states their "APIv4 specification is written following OpenAPI" on their docs page (https://www.gate.com/docs/developers/apiv4/en).
  - Their official SDK repos (gateio/gateapi-go, gateio/gateapi-python, gateio/gateapi-js, etc.) are all generated by OpenAPI Generator
  - The spec is NOT publicly hosted at a clean URL -- `https://www.gate.com/api/v4/openapi.json` returns `{"code":3,"message":"Error: invalid request"}`
  - The spec is likely served internally or embedded in their build pipeline
  - The `gateio/rest-v4` GitHub repo only contains a README.md

- **SDKs generated from spec** (the spec is embedded in these repos):
  - Go: https://github.com/gateio/gateapi-go
  - Python: https://github.com/gateio/gateapi-python
  - JavaScript: https://github.com/gateio/gateapi-js
  - Java: https://github.com/gateio/gateapi-java
  - PHP: https://github.com/gateio/gateapi-php
  - NodeJS: https://github.com/gateio/gateapi-nodejs
  - C#: (NuGet package)
  - The SDK repos may contain the spec as a source file or in their build artifacts

- **exchange-collection**: Status "WIP" for both REST and WS

- **Notes**: Gate.io almost certainly has an internal OpenAPI spec (they generate 7+ SDKs from it). Extracting it from one of the SDK repos or their docs build system would be valuable. Worth checking `gateapi-go/api_swagger_source.json` or similar files.

### 10. Bitstamp

- **Official OpenAPI spec**: YES

  - The page includes a "Download OpenAPI specification: Download" button
  - The spec is embedded in the page HTML (attribute `download="openapi.json"`)
  - The spec is NOT available at a standalone URL (common paths like `/api/openapi.json` return 404)
  - Format: OpenAPI 3.x (JSON)
  - Coverage: Public API v2 (full REST API)
  - Bitstamp also has a PSD2 Open Banking API at https://www.bitstamp.net/api-psd2

- **Notes**: This is a NEW import opportunity. The OpenAPI spec can be extracted from the Redoc page by loading it in a browser and capturing the download, or by parsing the embedded spec from the HTML. This would provide structured endpoint data for Bitstamp.

## Aggregator Projects

### openxapi (https://github.com/openxapi/openxapi)

- **Stars**: 9 | **License**: MIT | **Language**: Go
- **Approach**: Go program that automatically parses exchange API documentation and generates OpenAPI 3.0 and AsyncAPI 3.0 specifications
- **Exchange coverage** (as of Feb 2026):
  - Binance: REST (5 modules: spot, umfutures, cmfutures, options, pmargin) + WebSocket (AsyncAPI for all modules + streams)
  - OKX: REST only (200+ endpoints)
- **Quality**: High -- per-endpoint YAML files, Swagger UI preview, validation pipeline
- **Unique value**: Only source for Binance AsyncAPI specs and the most comprehensive Binance OpenAPI covering all sub-products (22+)
- **Update frequency**: Active; latest commit involves Binance WebSocket portfolio margin pro streams
- **SDKs generated**: Go, Python, JavaScript, Rust (Binance only)
- **Limitations**: Only 2 exchanges covered. OKX has no WebSocket specs yet.

## Gaps and Recommendations

### Priority 1: New imports with high value

1. **Bitstamp OpenAPI spec** -- Official spec available via Redoc download. Would replace agent-extracted endpoints with structured data. Action: Use Playwright to load https://www.bitstamp.net/api/ and capture the openapi.json download, then import via `import-openapi`.

2. **Gate.io OpenAPI spec** -- Extract from one of the official SDK repos (e.g., check `gateapi-go` or `gateapi-python` for a bundled spec file). Gate.io's SDKs are generated by OpenAPI Generator, so the source spec exists somewhere. Action: Search SDK repos for `openapi.json`, `swagger.yaml`, or `.openapi-generator` config files.

3. **Binance Postman collections** -- Cover ALL Binance products including the 9 newly added sections (options, margin_trading, wallet, copy_trading, portfolio_margin_pro, etc.). Action: Import from `binance-api-postman/collections/` for sections not yet covered by OpenAPI import.

### Priority 2: Supplementary imports

4. **KuCoin WebSocket specs** -- Official WebSocket OpenAPI specs available in `spec/ws/` directory of kucoin-universal-sdk. Would add structured WebSocket endpoint data.

5. **Hyperliquid community spec** -- Available from exchange-collection. Would provide structured endpoints to complement agent-extracted data.

6. **Upbit community Swagger** -- Available from uJhin/upbit-client. Would provide structured endpoints, though last updated mid-2024.

### Priority 3: Monitor for future availability

7. **openxapi expansion** -- Currently only covers Binance and OKX. Watch for new exchange support.

8. **exchange-collection completion** -- Bybit, KuCoin, HTX, Gate.io are "planned". Watch for completion.

9. **HTX, Crypto.com, Bitfinex, dYdX, Bithumb, Coinone, Korbit** -- No specs exist. Continue relying on agent-based markdown extraction.

### Not recommended

- **exchange-collection Binance/OKX specs** -- Redundant with openxapi (which has higher quality and more comprehensive coverage).
