# Forge Memory

Cross-session learnings. Minimum-signal gate: "Will a future agent act better knowing this?"
Keep under 3,000 tokens via aggressive deduplication.

## Architectural Decisions
- [2026-03-06] CCXT describe() data is cross-reference only — cite-only constraint prevents generating synthetic docs. Use ccxt_xref.py for validation, not doc generation.
- [2026-03-06] Single-page doc sites are correct behavior, not bugs: OKX (224K words), Gate.io (256K), HTX (325K across 4), BitMart (38K+34K), Crypto.com (35K), Bitstamp (22K), Korbit (25K).
- [2026-03-06] KuCoin uses opaque URL IDs (/docs-new/338210m0) — can't separate spot/futures by URL path. Merged into one section.

## Failed Approaches
- [2026-03-06] Generic crawl target prompt assumed 12 exchanges and suggested impractical sources (Discord, Wayback, forums, Google cache). These don't work for a deterministic crawl pipeline.

## Patterns Learned
- [2026-03-12] Skills that embed live store counts or exchange examples drift quickly as the dataset evolves. Re-run `store-report` and refresh scenario docs whenever endpoint/page coverage changes, especially in `xdocs-query` and its evaluation file.
- [2026-03-06] Scope_prefixes are critical for exchanges sharing one sitemap (Binance 9 sections, Coinbase 4 sections, Bitget 5 sections). Without them, URLs get claimed by the first section and skipped by the rest.
- [2026-03-06] render_mode: auto needed for JS-heavy doc sites (Coinbase, BitMart, most DEXes). Without it, link-follow only finds the seed page.
- [2026-03-06] 29 of 61 sections have 0 structured endpoints — potential OpenAPI/Postman import targets.
- [2026-03-06] CCXT describe() paths are relative to urls.api base, not absolute. Our DB stores full paths. Suffix-based matching (stripping /api/vN/ prefixes) is needed for cross-reference.
- [2026-03-06] Postman imports store {{url}}/{{host}} variable prefixes in endpoint paths. _normalize_path must strip all {{...}} patterns, not just {{url}}.
- [2026-03-06] NodePlaywrightFetcher.__init__ doesn't validate Node.js availability — cascade must call _find_node_pw_module() at selection time.
- [2026-03-06] revalidated_unchanged check in inventory_fetch.py short-circuits Playwright fallback — empty pages with matching hash won't get re-rendered.
- [2026-03-06] crawl4ai IS Playwright (hard dependency). crawl4ai can't render JS. No lightweight JS rendering alternative exists.
- [2026-03-06] LanceDB compaction: `compact_files()` + deprecated `cleanup_old_versions()` doesn't reduce fragments. Use `table.optimize(cleanup_older_than=timedelta(days=0))` instead — combines compact + prune + index optimize in one call. Reduced 3,568 → 9 fragments, 2.4GB → 908MB.
- [2026-03-08] _DOMAIN_MAP in semantic.py must be updated when new exchanges are registered. 11 exchanges were invisible to semantic search until domains were added. Always check after registering a new exchange.
- [2026-03-08] Incremental build with `--exchange` filter: stale detection must be scoped to the filtered exchange's pages only, or it deletes all other exchanges' chunks.
- [2026-03-08] Vector memory: Python float objects are 28 bytes each. A 1024d vector as Python list = ~32KB. 335K chunks = 10.24 GB heap. Must `row.pop("vector", None)` after each LanceDB `table.add(batch)`.
- [2026-03-08] eval_answer_pipeline.py: negative test entries (no expected URLs) must be excluded from retrieval quality metric denominators (MRR, nDCG, Hit rates) to avoid ~9.4% metric dilution.
- [2026-03-08] NEVER test dimension mismatch detection against production LanceDB index — it drops the table and creates a new one with different dimensions. Use a test fixture.
- [2026-03-08] v5-small uses only 1.27 GB VRAM on RTX 4070 Ti SUPER. OOM was from Python heap (10 GB vectors), not GPU memory. batch_size=16 is safe; 32 likely safe too.
- [2026-03-08] Golden QA: never use raw spec URLs (raw.githubusercontent, .yaml) as expected_urls — the pipeline suppresses these by design via `_is_spec_url()`. Use actual docs page URLs instead.
- [2026-03-09] BM25 IDF naturally penalizes overview/intro pages where query terms appear across many pages. Fix: `_apply_page_type_boost()` with URL-pattern matching (intro/overview/auth/rate-limit) and broad-question detection (how to, what is, etc.). +0.7% PFX, +0.6% question MRR with 1.4x boost factor.
- [2026-03-09] Section boost (rrf_score × 1.3 for section-matching URLs) hurts MRR by 0.5-1.1% in A/B benchmark. Root cause: promotes section-relevant pages that are less topically relevant over better cross-section results. Disabled by default.
- [2026-03-09] FlashRank vs Jina v3 in full pipeline: FlashRank +0.6% question MRR, Jina v3 +20% request_payload MRR, Jina v3 41% faster (0.83s vs 1.42s). Isolated reranker benchmark (M10) showed Jina v3 +15.6%, but position_aware_blend changes the dynamics. Jina v3 selected for balanced performance + speed.
- [2026-03-09] Always A/B benchmark each change independently, not all together. env vars: CEX_SECTION_BOOST, CEX_PAGE_TYPE_BOOST, CEX_RERANKER_BACKEND. Section boost looked good in theory but hurt in practice.
- [2026-03-08] Undocumented gate: must check `ep_count > 0` before gating (exchanges like Kraken have 0 stored endpoints). Use ALL-missing segment check (not ANY-missing) to avoid false negatives on partial path matches.
- [2026-03-09] Large single-page sites (OKX 225K, Gate.io 192K, HTX 130K words) have dense ToC/nav at the start (up to 16K bytes). `_make_excerpt` must skip nav regions or excerpts are just ToC entries. `_is_nav_region()` detects this via bullet/link density in a 1000-char window.
- [2026-03-09] Deprecated URL demotion (0.5× score for /abandoned-endpoints/, /deprecated/, /legacy-api/) hurts MRR by 3.5%. Root cause: KuCoin abandoned-endpoints have NO non-deprecated alternatives in results. Demotion punishes queries where deprecated is the only match. Needs conditional approach (only demote when non-deprecated alternatives exist).
- [2026-03-09] LanceDB fragment splitting: `table.optimize()` always re-merges all fragments into one. To control fragment size, must rewrite dataset via `lance.write_dataset(data, path, max_rows_per_file=N)` AFTER optimize. `compact_files(max_bytes_per_file=...)` only works when merging multiple existing fragments — it can't split a single fragment. Solution: compact first, then check size, then rewrite if needed.
- [2026-03-09] LanceDB runtime repo: solved via `compact_index(max_bytes_per_file=1_900_000_000)` which keeps fragments under GitHub LFS 2 GB limit. No need for HF Hub or R2 hosting. `sync_runtime_repo.py` applies this automatically.
- [2026-03-09] Runtime repo rebase: remote may have MacBook-made commits. Always `git fetch && git rebase origin/main` before pushing — never force-push unless no other option. Resolve conflicts by taking synced version (--theirs during rebase) for generated files.
- [2026-03-09] OPT research: `endpoint_search_text()` omits parameter names from FTS content. 94.4% of entries are under 100 chars. Adding params (from request_schema.parameters[].name) is the fastest quality win — no new tables, no prerequisites, ~15 LOC.
- [2026-03-09] Golden QA classification field is `classification` (not `expected_type`). 112 question, 29 endpoint_path, 30 error_message, 14 code_snippet, 15 request_payload.
- [2026-03-09] Binance OpenAPI import has 162 endpoints with unresolved `$ref` parameters — openapi_import.py stores the reference string but never resolves it to actual param names. Fix needed for OPT-1.
- [2026-03-09] code_snippet + request_payload failure root cause: `_generic_search_answer` uses `extract_search_terms` on raw code/JSON, producing poor FTS terms. The classifier already extracts method topics and exchange signatures, but only for post-hoc augmentation. Fix: pre-search query reformulation (OPT-11).
- [2026-03-09] Reranker landscape (2025-2026): GTE-reranker-modernbert-base (149M, 83.0% Hit@1) matches nemotron-rerank-1b (1.2B) and beats Jina v3 (81.33%). Standard CrossEncoder interface. Score-aware fusion (convex combination) beats RRF by +4.58% nDCG@10 on BEIR benchmarks.
- [2026-03-09] M18 sync_runtime_repo.py: directories without --hash-tree have no SHA256. _compare_manifests must fall back to files/bytes comparison, not always report CHANGED. CalVer tag collision: max_n must start at 0 to produce .1 after base tag.
- [2026-03-09] FIXED: answer.py had 3 compounding bugs — (1) `_search_pages` called with `exchange=` not `url_prefix=` (silent TypeError), (2) query_type_hint always "question" (dead code RRF weights), (3) BM25 shortcut included "question". Fix: code_snippet MRR 0.224→0.332 (+48%), nDCG@5 +1%.
- [2026-03-09] FIXED: position_aware_blend was no-op in semantic.py — `retrieval_score_key` defaulted to "rrf_score" but results had "score". Fix: pass `retrieval_score_key="score"`. Also had to update 2 semantic tests to use `rerank="never"` to isolate vector search from blend reordering.
- [2026-03-09] Spot check protocol v2: 8/8 pages matched live content perfectly across Binance, Bybit, OKX, KuCoin, Deribit, Hyperliquid, CoinEx, Gemini. DB integrity: 10,727 files, 0 missing, 0 orphaned. KuCoin worst for nav pollution (97%), OKX best (4%).
- [2026-03-09] Content quality gaps: dYdX 65% thin (283 pages), Kraken futures 49 pages all navigation (Docusaurus CSR), Bluefin 26% thin (login-gated), KuCoin param tables lost (ReadMe.io JS widgets render as `* * *`), 10 Gemini pages empty (JS redirects). 823 pages (7.7%) older than 7 days (Upbit, Paradex, Bitget, Coinbase, GMX, Perp).
- [2026-03-09] Reranker pipeline analysis: Jina v3 +15.6% MRR isolated (p=0.0014); 31.5% queries bypass reranker (direct routing + negative); 57% always reach it; controls 65% of final ranking for questions. Embedding upgrade (v5-nano→v5-small, +27.3% MRR) had 2x the reranker impact.
- [2026-03-09] Code_snippet failures: 57% caused by code syntax term pollution (ccxt, fetch_ticker, print, import, const). Tokens consume max_terms slots, AND query matches 0 pages, OR fallback matches everything. Fix: CODE_STOPWORDS set + use _search_pages_with_semantic in augmentation.
- [2026-03-09] Request_payload failures: 57% caused by missing operation-type inference. `{side, type, quantity, price}` = "place order" but pipeline searches raw param names. Fix: ~15-pattern operation map. Also exchange signature ordering bug: Gate.io's `{currency_pair}` matches before Bitstamp's `{type, currency_pair}`.
- [2026-03-10] A/B protocol: Always benchmark changes individually. M20 post-hoc A/B revealed bug2 (query_type_hint threading) was net negative (-0.66% MRR) because endpoint_path weights [1.5,0.5] were set theoretically, never validated. Fix: endpoint_path [0.7,1.3] (+5.2% MRR), code_snippet removed from BM25 shortcut (+3.5% MRR). Combined: MRR 0.6199→0.6308 (+1.8%).
- [2026-03-10] RRF weight validation: "question" vector-favoring weights [0.7,1.3] work well for most types. Only request_payload benefits from keyword-favoring [1.3,0.7]. endpoint_path was HURT by keyword-favoring [1.5,0.5]. Always A/B test weight changes per type.
- [2026-03-10] FTS5 AND operators (literal "AND" token) get passed as semantic query text to the embedder. _search_pages_with_semantic should accept separate fts_query and semantic_query params. Medium impact on embedding quality.
- [2026-03-10] Reranker landscape (2026-03): Jina v3 still BEIR leader (61.94 nDCG@10) under 2B params. GTE-ModernBERT-base (149M, BEIR 56.73, CoIR 79.99) worth benchmarking for code queries. Score-aware linear fusion beats RRF by ~4.58% nDCG (TopK benchmark, BEIR datasets). Contextual AI Reranker v2 has instruction-following (could replace heuristic URL demotion).
- [2026-03-10] Auto-rerank threshold (12 candidates) too high for small exchanges (backpack 1 page, nado 192 pages). Should lower to 6 or use rerank="always" in answer pipeline.
- [2026-03-10] CC score-aware fusion A/B'd: neutral vs RRF on 206 queries (MRR -0.47%, error_message PFX -3.70%, code_snippet MRR +2.59%). Root cause: reranker only triggers ~57% of queries — without it, raw LanceDB distance doesn't carry more info than rank. RRF stays default. CC available via CEX_FUSION_MODE=cc.
- [2026-03-10] Score erasure in answer.py: semantic_search returns `score`, `blended_score`, `rerank_score` but _search_pages_with_semantic discarded all of them when re-fetching from SQLite. Fixed: now preserves `semantic_score` field. Infrastructure for future score-aware features.
- [2026-03-10] position_aware_blend inverts distance-based scores (pre-existing bug): for query_type="vector", `score` is _distance (lower=better), but blend max-normalizes it treating higher=better. Partially mitigated by reranker re-sort.
